Knowledge System

4-Layer 지식 시스템

에이전트 지식은 4개 계층으로 구성됩니다. 하위 계층은 범용, 상위 계층은 프로젝트 특화.

자세히 보기 →
L0
Common Sense
core/knowledge/common-sense.yaml
모든 에이전트가 공유하는 범용 상식. 논리 규칙, 분석 원칙, 언어 규범. 도메인과 무관하게 항상 로드되는 최하위 레이어.
L1
역할 지식
.claude/agents/*.md, leaves/**/*.md
에이전트 역할 정의, 분석 프레임워크, 출력 포맷 규칙. 각 Division Lead와 Leaf 에이전트의 전문 역량 정의.
L2
도메인 축적 지식
domains/{domain}/knowledge/
산업별 특화 지식. frameworks.md, data-sources.md, agents/. domains/example/을 복사해 새 도메인 플러그인 생성 가능.
L3
프로젝트 맥락
{project}/
1회성 프로젝트 실행 데이터. 00-client-brief.md, hypotheses.yaml, findings/, sync/, thinking-loop/, reports/, qa/. 골든 팩트는 findings/golden-facts.yaml에 SSOT로 관리.
출력 포맷 — 4-Layer 피라미드
L0
Claim
최상위 주장
하나의 명확한 주장 문장. SCR 스토리라인의 Resolution.
L1
Evidence
근거
Claim을 지지하는 2~3개 핵심 근거. 논리적 연결 필수.
L2
Data
수치 데이터
Golden Facts에서 인용된 검증된 수치. VL-1~3 검증 완료.
L3
Source
출처
URL, 문서명, 날짜. source-url-verifier로 유효성 검증.
API Guide

데이터 수집 우선순위

에이전트는 아래 우선순위에 따라 데이터를 수집합니다. 상위 수단이 가능하면 항상 우선 사용.

자세히 보기 →
순위 수단 용도 비고
1 API (직접 호출) DART, FRED, ECOS, 공공 데이터 포털 등 구조화 데이터 scripts/api-caller.py 래퍼 사용
2 Exa Search 신뢰도 높은 웹 검색. 최신 기사, 보고서, 논문 API 키 필요. .env EXASEARCH_API_KEY
3 WebSearch 일반 웹 검색. 공개 정보, 뉴스 Claude 기본 내장 도구
4 WebFetch / Firecrawl 특정 URL 크롤링. 구조화된 HTML 파싱 도메인 권한 필요. settings.local.json 등록
주요 데이터 소스
API / 소스 도메인 데이터 유형 환경변수
DART dart.fss.or.kr 한국 기업 공시, 재무제표 DART_API_KEY
FRED fred.stlouisfed.org 미국 거시경제 지표 FRED_API_KEY
ECOS (한국은행) ecos.bok.or.kr 한국 경제 통계 ECOS_API_KEY
Exa Search exa.ai 고품질 웹 검색 EXASEARCH_API_KEY
Firecrawl firecrawl.dev 웹 크롤링, 구조화 파싱 FIRECRAWL_API_KEY
Commands

명령어 레퍼런스

Deep-Briefing 진입점 및 주요 스크립트. 복사 버튼으로 바로 사용할 수 있습니다.

자세히 보기 →
/setup — 초기 환경 설정 
# Claude Code 실행 후 슬래시 명령어로 진입
claude

# 환경 점검 + 도메인 생성 + API 키 설정 (인터랙티브)
> /setup
/research — 리서치 시작 
# Interactive 모드 (매 단계 사용자 게이트)
> /research interactive {project-name} {주제}

# Auto 모드 (직행, 가설 자동 확정)
> /research auto {project-name} {주제}

# Team 모드 (Deep Discovery + 전방위 토론)
> /research team {project-name} {주제}
spawn-leads.sh — Lead CLI 병렬 실행 
# tmux N-pane Lead CLI 실행 (Division 동적 감지)
./scripts/spawn-leads.sh {project-name}

# attach 모드 (실행 후 tmux 세션 자동 연결)
./scripts/spawn-leads.sh {project-name} --attach

# auto 모드 (모든 권한 자동 승인)
./scripts/spawn-leads.sh {project-name} --auto
compile-lead-context.sh — Lead 컨텍스트 컴파일 
# PM이 Phase 0 후 실행. Division Lead에게 프로토콜/도메인 컨텍스트 전달
./scripts/compile-lead-context.sh {project-name}

# Phase 2 지시 자동 전송 (활성 Lead CLI에)
./scripts/send-phase2.sh {project-name}

# 임의 메시지 전송
./scripts/send-to-leads.sh {project-name} "메시지 내용"
유틸리티 스크립트 
# 프로젝트 디렉토리 스캐폴딩
./scripts/init-project.sh {project-name}

# findings → 차트 자동 생성
python scripts/generate-charts.py {project-name}

# Golden Facts 교차 검증
python scripts/verify-facts.py {project-name}

# API 키 유효성 점검
./scripts/check-api-keys.sh
