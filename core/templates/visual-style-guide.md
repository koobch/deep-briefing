# 시각 스타일 가이드 (Visual Style Guide)

> 보고서 차트 및 슬라이드의 시각적 일관성을 위한 기준.
> python-pptx 자동 생성 및 수동 PPT 작성 시 모두 적용한다.

## 설계 원칙

1. **명확성 우선**: 장식보다 메시지 전달. 불필요한 3D, 그림자, 그라데이션 제거
2. **데이터-잉크 비율 최대화**: Tufte 원칙 — 데이터를 표현하는 잉크 외에는 최소화
3. **1차트 = 1메시지**: 차트 하나가 전달하는 핵심 메시지는 1개
4. **접근성**: 색각 이상자도 구분 가능한 색상 + 패턴 병용

## 색상 팔레트

### 주 색상 (Primary)
| 용도 | 색상 | HEX | 사용처 |
|------|------|-----|--------|
| 핵심 강조 | 딥 블루 | `#2E86AB` | 주요 데이터, 핵심 메시지 |
| 보조 강조 | 버건디 | `#A23B72` | 비교 대상, 경쟁사 |
| 경고/위험 | 레드 | `#C73E1D` | DOWNSIDE, 리스크, 감소 |
| 긍정/성장 | 틸 | `#44BBA4` | UPSIDE, 기회, 증가 |
| 중립 | 슬레이트 | `#5D6D7E` | 기준선, 평균, 배경 데이터 |

### 시나리오 색상
| 시나리오 | HEX | 비고 |
|----------|-----|------|
| BASE | `#2E86AB` | 주 색상 |
| UPSIDE / BULL | `#44BBA4` | 긍정 |
| DOWNSIDE / BEAR | `#C73E1D` | 경고 |

### 순차 색상 (다범주 비교 시)
```
#2E86AB → #A23B72 → #F18F01 → #C73E1D → #3B1F2B → #44BBA4 → #5D6D7E → #AF7AC5
```
- 최대 8개까지. 그 이상은 그룹핑 후 "기타"로 합산

## 타이포그래피

### 폰트 우선순위
| 환경 | 1순위 | 2순위 | 3순위 |
|------|-------|-------|-------|
| macOS/PPT | Pretendard | Apple SD Gothic Neo | AppleGothic |
| Windows | Pretendard | Malgun Gothic | NanumGothic |
| 차트 (matplotlib) | Pretendard | AppleGothic | sans-serif |

### 폰트 크기 기준
| 요소 | 차트 (PNG) | 슬라이드 (PPT) |
|------|-----------|---------------|
| 차트/슬라이드 타이틀 | 14pt bold | 24pt bold |
| 축 레이블 | 10pt | 14pt |
| 데이터 레이블 | 9pt | 12pt |
| 출처 각주 | 8pt | 10pt italic |
| 본문 텍스트 | — | 16pt |

## 차트 디자인 규칙

### 공통 규칙
- 배경: 순백 `#FFFFFF` (dark 모드 제외)
- 격자선: 회색 `#CCCCCC`, 투명도 30%, 점선
- 상단/우측 테두리(spine) 제거
- 모든 차트에 데이터 레이블 직접 표기 (범례보다 직접 레이블 우선)
- 출처 각주: 차트 하단 좌측, `Source: [S##] {소스명}, {시점}` 형식

### 차트 유형별 가이드

| 차트 유형 | 최적 사용 | 주의사항 |
|-----------|----------|----------|
| 꺾은선 (Line) | 시계열 추이 (3개 시리즈 이하) | 4개+ 시리즈는 그룹 막대로 전환 |
| 막대 (Bar) | 범주 비교 | 값 기준 내림차순 정렬. 레이블 길면 수평 |
| 그룹 막대 (Grouped Bar) | 다차원 비교 | 최대 4그룹. 그 이상은 소규모 다중 차트 |
| 파이/도넛 (Pie) | 구성 비율 (합계 100%) | 최대 6조각. 그 이상은 "기타" 합산 |
| 워터폴 (Waterfall) | 분해 구조 (매출→비용→이익) | 증가=블루, 감소=레드, 합계=슬레이트 |
| 시나리오 (Scenario Bar) | BASE/UPSIDE/DOWNSIDE 비교 | 시나리오 색상 규칙 적용 |
| 히트맵 (Heatmap) | 2차원 매트릭스 | 셀 값 직접 표기 필수 |
| 마리메꼬 (Marimekko) | 시장 구성 (세그먼트 × 비중) | X축 폭 = 세그먼트 크기, Y축 = 구성 비율 |
| 하비볼 (Harvey Ball) | 정성 평가 매트릭스 | ●◕◑◔○ 5단계. 색상 아닌 채움으로 구분 |

### Exhibit 포맷
```
Exhibit {N}: {Action Title — 차트가 전달하는 메시지 문장}

[차트]

Source: [S##] {소스명} ({시점}) | Note: {보충 설명}
```

## 슬라이드 레이아웃 원칙

### 레이아웃 유형
| layout | 설명 | 사용처 |
|--------|------|--------|
| `title_slide` | 타이틀 + 서브타이틀 | 표지, 섹션 구분 |
| `content` | Action Title + 본문 | 분석 결과 |
| `two_column` | 좌/우 분할 | 비교, 대조 |
| `chart` | Action Title + 차트 1개 | 데이터 시각화 |
| `table` | Action Title + 테이블 | 상세 비교 |
| `summary` | 핵심 메시지 + 넘버링 | 요약, Next Steps |

### 여백 및 정렬
- 슬라이드 마진: 상하좌우 최소 1인치
- 텍스트/차트 좌측 정렬 기본 (중앙 정렬은 타이틀 슬라이드만)
- 슬라이드 하단에 페이지 번호 + 기밀 등급 표시 영역 확보
