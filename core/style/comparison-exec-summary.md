### 핵심 발견: BCG의 3행은 슬라이드 하단을 채우지 않음!

```
BCG 레퍼런스 실측:
  헤더: 0→60 (60px)
  Row1:  62→315 (253px)
  Row2: 315→545 (230px)
  Row3: 545→740 (195px)
  빈 공간: 740→1040 (300px!) ← 하단 28%가 비어있음
  Source: 1040→1060 (20px)

→ 3행의 총 높이 = 678px (전체의 63%)
→ 행이 균등 분배가 아니라 콘텐츠 높이에 따라 다름
→ Row1(253) > Row2(230) > Row3(195)
```

### 핵심 문제: 본문 폰트가 11px여야 하는데 14px

BCG 레퍼런스에서 한 줄에 들어가는 글자 수로 역산:
```
"The global market outlook for 2025 remains relatively steady, with declining optimism in 2025..."
→ 이 문장이 x:190 → x:1780 (1590px) 안에 한 줄로 들어감
→ 영문 기준 약 110자
→ 11px 폰트에서 평균 글자폭 ~7px → 110 * 7 = 770px (행 폭의 48%)
→ 실제로는 문장이 더 길어서 1줄 반 정도 차지
→ 따라서 11-12px가 맞음

프로토타입 14px에서는:
→ 같은 문장이 더 넓어져서 행을 더 많이 차지
→ 결과적으로 텍스트가 적어 보이고 빈 공간이 많아짐
```
